前篇回顾：Git分布式版本管理系统入门（一）

五、Github开源代码库工作方式

想要完成一个简单的项目，下图中最基本的操作必须要掌握。
首先我们要知道，一个项目代码仓库是由任意个分支组成的，每个仓库都应该会自动初始化一个主支【master】。本地仓库有分支，远程仓库也有分支。
分支的含义：其实是主支的分支，就像树枝一样。新建一个分支时，Git会自动把主支的内容copy一份到你的分支。

  • Git分布式版本管理系统日常习惯：
  1. 主支【master】的作用：把分支合并进主支
  2. 本地仓库除了【master】主支外，习惯性地还会新建一个自己的分支(一次开发完后习惯性地删除自己的分支)。主支用于查看项目代码，自己的分支用于修改项目。(非必要)
  3. 本地仓库推送到远程仓库时不应该直接推送到远程主支【master】，应新建一个自己的分支，把项目推送到自己的分支。(例如下图的【xiaoming】分支)
  4. 合并分支必须通过项目管理员同意。

  • 情景：
  1. 假如你叫小明，你刚刚部署了本地Git仓库(Local)，Git就会自动帮你新建一个本地主支【master】(远程仓库也一样)。
  2. 你已经把远程仓库连接好，并且在远程仓库上新建了两个分支【xiaoming】和【partner】，本地新建了一个分支【xiaoming】。
  3. 切换到本地【xiaoming】分支，这时你就可以使用克隆命令或者拉取命令把远程仓库主支【master】的项目拉取到本地仓库(如果你的远程仓库有项目的话)。
  4. 好了，拉取到本地仓库后你就可以开始开发你的项目，在你的项目上面做修改。
  5. 开发完成！你想把项目推送到远程仓库合并进主支。现在可以打开命令行用推送命令把你的项目推送到远程仓库中自己的分支【xiaoming】。
  6. 推送成功！下一步便是把自己的分支合并进远程主支(Github网页操作)。这里就需要项目的管理员去审核你的修改，管理员审核完后就可以把你的分支合并，合并后你修改的内容就会出现在远程主支了。(如出现严重冲突要拉回本地修改，简单的冲突可以在Github网页修改)。
  7. 最后一步，把远程主支的项目拉取到本地仓库的【master】主支，删除本地【xiaoming】主支，一次的开发完成了。

六、Git的基本工作流程

  1. git clone克隆远程资源到本地目录，作为工作目录；
  2. 然后在本地的克隆目录上添加或修改文件；
  3. (如果远程修改了，需要同步远程的内容，直接git pull就可以更新本地的文件；)
  4. 本地在修改之后，可以通过git status 查看修改的文件。然后使用git add 添加修改的文件暂到缓冲区；
  5. 在添加之后，可以使用git commit添加到当前的工作区；
  6. 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交；
  7. git push将本地的修改推送到远程的git服务器。

八、配置SSH公私钥，使用SSH连接到Github

  1. 配置SSH公私钥有什么用？
    我们要知道，当你使用https的连接去推送项目时，在每次推送项目到远程时都需要输入账号密码验证你的身份。
    而SSH公私钥代表你的身份，可以把公私钥粘贴到N个git的服务器上（网站，云服务器），之后推送项目就不需重复输入密码验证身份了。
    查看官方文档：生成新SSH密钥以用于身份验证
